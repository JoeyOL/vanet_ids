## 联邦训练改进证据与性能验证摘要

### 1. 实验说明

- **改进方案输出目录**：`outputs/`
- **基线方案输出目录**：`outputs/baseline_fedavg/`
- **复现实验数据**：`data/demo_vehicular.csv`
- **说明**：当前结果用于验证工程链路、报告机制与指标采集能力；论文正式结论建议在真实 `VeReMi / Car-Hacking` 数据上重复至少 3 次并汇报均值与标准差。

### 2. 实验配置

| 方案 | 客户端参与率 | Top-K | 量化位宽 | 说明 |
| --- | --- | --- | --- | --- |
| 改进方案 | 0.75 | 0.2 | 8 bit | 动态节点选择 + 稀疏化 + 量化 |
| 基线方案 | 1.0 | 1.0 | 32 bit | 全节点参与 + 无压缩 |

### 3. 关键结果对比

| 指标 | 改进方案 | 基线方案 | 对比结论 |
| --- | ---: | ---: | --- |
| 总训练时间（ms） | 933.24 | 1063.74 | 改进方案下降约 12.27% |
| 平均轮次耗时（ms） | 304.51 | 332.83 | 改进方案下降约 8.51% |
| 总通信量（bytes） | 79,848 | 404,928 | 改进方案下降约 80.28% |
| 报告内通信压缩率 | 0.7371 | 0.0000 | 改进方案相对本轮原始上传减少 73.71% |
| 平均隐私代理分数 | 0.9211 | 0.7000 | 改进方案更高 |
| Accuracy | 0.60 | 0.60 | 两者持平 |
| Recall | 1.00 | 1.00 | 两者持平 |
| F1-score | 0.75 | 0.75 | 两者持平 |
| 推理延迟（ms） | 3.45 | 3.14 | 差异很小 |

### 4. 结果解读

- **通信侧**：改进方案通过 `Top-K=0.2` 与 `8bit` 量化，将总通信量从 `404,928 bytes` 降到 `79,848 bytes`，说明“参数压缩”改进是有效的。
- **训练侧**：在本次小规模实验中，总训练时间与平均轮次耗时均低于基线，说明“节点选择 + 降低参与客户端数”对系统吞吐有正向作用。
- **检测侧**：当前小样本实验下，`Accuracy / Recall / F1` 与基线保持一致，表明压缩与筛选并未引入额外性能退化。
- **隐私侧**：由于原始数据始终不出本地，且改进方案进一步减少了上传更新量，因此隐私代理分数高于基线。
