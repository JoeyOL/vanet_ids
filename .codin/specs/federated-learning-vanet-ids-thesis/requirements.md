# Requirements Document

## Introduction

本文档定义本科毕业设计论文《基于联邦学习的车联网入侵检测系统设计与实现》完整初稿的写作需求。论文初稿应以现有开题报告、任务书、项目代码及仓库内可获取的模板材料为基础，形成可继续润色和套用学校格式的完整正文。论文目标篇幅为 20000 字以上，优先采用学术规范且自然的表达方式，避免明显模板化痕迹。参考文献优先从开题报告中提取与整理。

## Glossary

- **论文初稿**：指本次需要产出的完整毕业设计论文正文草稿，包含题名、摘要、关键词、目录、正文各章、总结与展望、参考文献等内容。
- **开题报告**：指用户已具备的毕业设计开题材料，用于提炼研究背景、研究目标、研究内容、技术路线和参考文献线索。
- **任务书**：指毕业设计任务说明材料，用于界定论文范围、研究目标和成果要求。
- **项目代码**：指当前仓库中与车联网入侵检测系统相关的实现代码，用于支撑系统设计、模块实现和实验过程描述。
- **学校模板**：指学校或学院要求的毕业论文排版或章节模板材料。
- **联邦学习**：指多参与方在不直接共享原始数据的条件下联合训练模型的方法。
- **车联网**：指车辆、路侧单元、网络基础设施及相关平台互联形成的网络环境。
- **入侵检测系统**：指用于识别异常流量、恶意行为或攻击事件的分析与检测系统。

## Requirements

### Requirement 1

**User Story:** 作为本科毕设学生，我希望先形成与题名一致的完整论文结构与基础内容，以便快速得到可继续修改的全文初稿。

#### Acceptance Criteria

1. 当用户提供论文题名时，论文初稿应该输出与题名一致的摘要、关键词和章节名称。
2. 在本科计算机类毕业设计场景下，论文初稿应该包含摘要、关键词、绪论、相关技术、系统分析与设计、系统实现、实验与结果分析、总结与展望、参考文献等核心部分。
3. 如果学校模板在仓库中可获取，那么论文初稿应该按模板要求组织章节层级和主要组成部分。
4. 如果学校模板暂不可获取，那么论文初稿应该先按通用本科毕设结构生成完整正文内容。

### Requirement 2

**User Story:** 作为本科毕设学生，我希望论文正文能够准确吸收开题报告和任务书中的研究目标与研究内容，以便保证论文主题一致且不偏题。

#### Acceptance Criteria

1. 当存在开题报告时，论文初稿应该提炼研究背景、研究目的、研究意义和研究内容并映射到绪论章节。
2. 当存在任务书时，论文初稿应该提炼任务目标、成果范围和约束条件并映射到论文整体结构中。
3. 在开题报告与任务书对研究范围有明确限定的情况下，论文初稿应该围绕限定范围组织正文，不扩展到无关主题。
4. 如果开题报告中包含参考文献线索，那么论文初稿应该优先基于这些线索组织参考文献部分。

### Requirement 3

**User Story:** 作为本科毕设学生，我希望论文中的系统设计与实现章节能够对应当前项目代码，以便正文内容具备工程依据。

#### Acceptance Criteria

1. 当项目代码可读取时，论文初稿应该将系统模块、数据流程、训练流程和检测流程映射到系统设计与实现章节。
2. 当项目代码中存在联邦学习、数据处理、模型训练或检测推理相关实现时，论文初稿应该说明其职责、输入输出和协作关系。
3. 在代码结构能够反映系统架构的情况下，论文初稿应该据此描述系统总体架构与关键模块设计。
4. 如果代码中缺少某些论文所需细节，那么论文初稿应该以不违背现有实现的方式进行概括描述，并避免编造具体实现结果。

### Requirement 4

**User Story:** 作为本科毕设学生，我希望论文中的实验与分析章节尽量基于已有项目材料展开，以便形成可信的研究论证。

#### Acceptance Criteria

1. 当仓库或已有材料中存在实验配置、数据集信息、评价指标或实验结果时，论文初稿应该据此组织实验环境、实验方案、结果展示和结果分析内容。
2. 在已有材料能够支撑对比分析的情况下，论文初稿应该描述模型效果、检测性能或联邦学习方案的分析结论。
3. 如果缺少完整实验结果，那么论文初稿应该明确保留待补充位置，并优先写出可复现的实验设计与分析框架。
4. 如果开题报告对实验目标有明确描述，那么论文初稿应该使实验章节与这些目标保持一致。

### Requirement 5

**User Story:** 作为本科毕设学生，我希望整篇论文表达学术规范且尽量自然，以便后续修改成本更低并降低机械化痕迹。

#### Acceptance Criteria

1. 在整篇论文写作过程中，论文初稿应该采用学术规范、保守稳妥且自然连贯的中文表达。
2. 当章节涉及系统设计或实现时，论文初稿应该优先使用工程化叙述而非空泛表述。
3. 在实验与分析章节中，论文初稿应该使用围绕现有材料展开的论证方式，而不是脱离项目的泛化描述。
4. 如果参考文献需要整理，那么论文初稿应该优先从开题报告中提取并组织参考文献内容。
